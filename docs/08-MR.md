# MR — Modelo Relacional (físico, DDL-ready) — NeatSpace
### Tomo Técnico VIII · De entidades a tablas PostgreSQL/PostGIS

**Base:** `docs/07-MER.md` (conceptual), `specs/schemas/entities.yaml` y los deep-dives 03/04/05. Este documento es **DDL-ready para PostgreSQL 15+ con PostGIS**: tipos, claves, `CHECK`, `UNIQUE`, FKs, índices (incl. GiST geográfico y el índice de equidad de NeatMatch) y las reglas **append-only** / **hash-chain** / **partida doble balanceada**. Materializa las invariantes IN-1..8 del MER.

> **Convenciones.** PK `uuid` (anti-enumeración/IDOR, RN-7); montos `integer` en **CLP** (sin decimales); marcas de tiempo `timestamptz`; geo en **SRID 4326**. Lo que la base **no** puede garantizar (reglas multi-tabla, orquestación) se marca como *validación en aplicación*.

## 1. Extensiones y utilidades

```sql
CREATE EXTENSION IF NOT EXISTS postgis;
CREATE EXTENSION IF NOT EXISTS pgcrypto;   -- gen_random_uuid()

-- Cuenta argumentos no nulos (para el CHECK XOR de billetera, IN-1)
CREATE OR REPLACE FUNCTION num_nonnull(VARIADIC anyarray)
RETURNS int LANGUAGE sql IMMUTABLE AS $$
  SELECT count(*)::int FROM unnest($1) x WHERE x IS NOT NULL;
$$;

-- Guardia append-only: bloquea UPDATE/DELETE (IN-3)
CREATE OR REPLACE FUNCTION prevent_mutation()
RETURNS trigger LANGUAGE plpgsql AS $$
BEGIN
  RAISE EXCEPTION 'Tabla append-only %: operación % no permitida', TG_TABLE_NAME, TG_OP;
END $$;
```

## 6. NeatWallet (dinero, partida doble)

```sql
CREATE TABLE neatwallet (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tipo        wallet_tipo NOT NULL,
  usuario_id  uuid UNIQUE REFERENCES usuario(id),
  empresa_id  uuid UNIQUE REFERENCES empresa(id),
  rol_sistema wallet_rol_sistema,
  moneda      char(3) NOT NULL DEFAULT 'CLP',
  creado_en   timestamptz NOT NULL DEFAULT now(),
  -- IN-1: identidad XOR — exactamente uno de los tres
  CONSTRAINT wallet_identidad_xor CHECK (num_nonnull(usuario_id, empresa_id, rol_sistema) = 1),
  -- coherencia tipo↔columna
  CONSTRAINT wallet_tipo_coherente CHECK (
    (tipo='usuario' AND usuario_id IS NOT NULL) OR
    (tipo='empresa' AND empresa_id IS NOT NULL) OR
    (tipo='sistema' AND rol_sistema IS NOT NULL)
  ),
  -- las cuentas de sistema son únicas por rol (un solo ESCROW, una sola COMISION, ...)
  CONSTRAINT wallet_rol_sistema_uq UNIQUE (rol_sistema)
);
ALTER TABLE empresa ADD CONSTRAINT empresa_wallet_fk
  FOREIGN KEY (wallet_id) REFERENCES neatwallet(id);

CREATE TABLE pago (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  acuerdo_id    uuid REFERENCES acuerdo(id),
  mp_payment_id text,                                   -- id en MercadoPago
  estado        pago_estado NOT NULL DEFAULT 'pendiente',
  monto         integer NOT NULL CHECK (monto > 0),
  creado_en     timestamptz NOT NULL DEFAULT now()
);

CREATE TABLE mercadopago_event (                        -- APPEND-ONLY: idempotencia del webhook
  id          text PRIMARY KEY,                         -- event-id de MercadoPago (dedup)
  pago_id     uuid REFERENCES pago(id),
  payload     jsonb NOT NULL,
  procesado_en timestamptz NOT NULL DEFAULT now()
);

CREATE TABLE transaccion (                              -- APPEND-ONLY: agrupador balanceado
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tipo          tx_tipo NOT NULL,
  referencia_dominio text,                              -- servicio_id | pago_id | retiro_id
  idempotency_key text UNIQUE NOT NULL,                 -- RN-1 (400 si falta, en API)
  creado_en     timestamptz NOT NULL DEFAULT now()
);

CREATE TABLE ledger_entry (                             -- APPEND-ONLY: el asiento
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  transaccion_id uuid NOT NULL REFERENCES transaccion(id),
  wallet_id     uuid NOT NULL REFERENCES neatwallet(id),
  direccion     ledger_dir NOT NULL,
  monto         integer NOT NULL CHECK (monto > 0),
  concepto      text,
  creado_en     timestamptz NOT NULL DEFAULT now()
);
```

**IN-2 · Transacción balanceada** (Σ débitos = Σ créditos), como *constraint trigger* diferido al commit:

```sql
CREATE OR REPLACE FUNCTION check_tx_balance()
RETURNS trigger LANGUAGE plpgsql AS $$
DECLARE d bigint; c bigint; n int;
BEGIN
  SELECT
    coalesce(sum(monto) FILTER (WHERE direccion='debito'),0),
    coalesce(sum(monto) FILTER (WHERE direccion='credito'),0),
    count(*)
  INTO d, c, n
  FROM ledger_entry WHERE transaccion_id = NEW.transaccion_id;
  IF n < 2 OR d <> c THEN
    RAISE EXCEPTION 'Transacción % desbalanceada: débitos=% créditos=% asientos=%',
      NEW.transaccion_id, d, c, n;
  END IF;
  RETURN NULL;
END $$;

CREATE CONSTRAINT TRIGGER tx_balance_chk
  AFTER INSERT ON ledger_entry
  DEFERRABLE INITIALLY DEFERRED
  FOR EACH ROW EXECUTE FUNCTION check_tx_balance();
```

> **Saldo = proyección.** No hay columna `saldo`: `saldo(w) = Σ créditos − Σ débitos` (vista/consulta). Para billeteras `usuario`/`empresa` la aplicación exige `saldo ≥ 0`; las de `sistema` admiten negativo (esperado en ESCROW/RECUPERO).

## 7. Reputación

```sql
CREATE TABLE evaluacion (
  id           uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  servicio_id  uuid NOT NULL REFERENCES acuerdo(id),    -- servicio ≡ acuerdo (nota MER §2)
  evaluador_id uuid NOT NULL REFERENCES usuario(id),
  evaluado_id  uuid NOT NULL REFERENCES usuario(id),
  estrellas    smallint NOT NULL CHECK (estrellas BETWEEN 1 AND 5),
  atributos    jsonb NOT NULL DEFAULT '{}',             -- {amable:true, rapido:true, ...}
  comentario   text,                                    -- moderado + PII-limpio antes de visible
  visible      boolean NOT NULL DEFAULT false,          -- double-blind (único campo mutable)
  creado_en    timestamptz NOT NULL DEFAULT now(),
  UNIQUE (servicio_id, evaluador_id),                   -- IN-4
  CHECK (evaluador_id <> evaluado_id)
);

CREATE TABLE reputation_log (                           -- APPEND-ONLY encadenado por hash (IN-3, IN-8)
  id           uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  usuario_id   uuid NOT NULL REFERENCES usuario(id),
  evento       reput_evento NOT NULL,
  payload      jsonb NOT NULL,
  evaluacion_id uuid REFERENCES evaluacion(id),
  hash_prev    bytea,
  hash_actual  bytea NOT NULL,                          -- H(hash_prev || payload)
  creado_en    timestamptz NOT NULL DEFAULT now()
);
CREATE INDEX ON reputation_log (usuario_id, creado_en);

CREATE TABLE trust_score (                              -- PROYECCIÓN recomputable (IN-7)
  neatprofile_id uuid PRIMARY KEY REFERENCES neatprofile(id) ON DELETE CASCADE,
  valor_0_100    smallint NOT NULL DEFAULT 0 CHECK (valor_0_100 BETWEEN 0 AND 100),
  valor_bayesiano numeric(6,4),
  atributos_0_100 jsonb NOT NULL DEFAULT '{}',
  n_evaluaciones integer NOT NULL DEFAULT 0,
  nivel_verificacion smallint NOT NULL DEFAULT 0,
  recalculado_en timestamptz NOT NULL DEFAULT now()
);
```

```sql
-- Feed de oportunidades en vivo (GiST geográfico + filtros)
CREATE INDEX idx_opp_geo   ON oportunidad USING gist (geo_aprox);

CREATE INDEX idx_trust_rank ON trust_score (valor_0_100 DESC);

CREATE INDEX idx_impr_window ON match_impression (profesional_id, categoria_id, creado_en DESC);

CREATE INDEX idx_ledger_wallet ON ledger_entry (wallet_id, creado_en DESC);

```

```sql
CREATE TRIGGER ao_aceptar   BEFORE UPDATE OR DELETE ON acuerdo_aceptacion
  FOR EACH ROW EXECUTE FUNCTION prevent_mutation();
CREATE TRIGGER ao_offer     BEFORE UPDATE OR DELETE ON price_offer
  FOR EACH ROW EXECUTE FUNCTION prevent_mutation();
CREATE TRIGGER ao_replog    BEFORE UPDATE OR DELETE ON reputation_log
  FOR EACH ROW EXECUTE FUNCTION prevent_mutation();
CREATE TRIGGER ao_urgente   BEFORE UPDATE OR DELETE ON aceptacion_urgente
  FOR EACH ROW EXECUTE FUNCTION prevent_mutation();
CREATE TRIGGER ao_impr      BEFORE UPDATE OR DELETE ON match_impression
  FOR EACH ROW EXECUTE FUNCTION prevent_mutation();
CREATE TRIGGER ao_mpevent   BEFORE UPDATE OR DELETE ON mercadopago_event
  FOR EACH ROW EXECUTE FUNCTION prevent_mutation();

  FOR EACH ROW EXECUTE FUNCTION prevent_mutation();   -- solo DELETE (checker_id se completa 1 vez, en app)
```

> **Excepción `evaluacion`:** no es append-only total — el único cambio permitido es `visible: false→true` en la publicación *double-blind*. Se restringe con un trigger que rechaza cualquier UPDATE que toque otra columna (validación en aplicación + trigger fino).

## 11. Otras reglas de integridad

```sql
-- IN-5: oportunidad.tipo inmutable
CREATE OR REPLACE FUNCTION opp_tipo_inmutable()
RETURNS trigger LANGUAGE plpgsql AS $$
BEGIN
  IF NEW.tipo <> OLD.tipo THEN
    RAISE EXCEPTION 'oportunidad.tipo es inmutable (IN-5)';
  END IF;
  RETURN NEW;
END $$;
CREATE TRIGGER opp_tipo_guard BEFORE UPDATE ON oportunidad
  FOR EACH ROW EXECUTE FUNCTION opp_tipo_inmutable();
```

**Validaciones que viven en la aplicación** (la base no las puede garantizar sola):

| Regla | Por qué en app |
|---|---|
| `Idempotency-Key` presente en toda escritura de dinero (RN-1) | Es de capa HTTP; la BD solo garantiza unicidad de la clave. |
| Comisión = `round(total×0.20)` server-side; `neto = total − comisión` (RN-2) | Cálculo de negocio; la BD guarda el resultado. |
| `release` solo con entrega confirmada o ventana vencida (RN-4) | Orquestación de estados multi-tabla. |
| Webhook = fuente de verdad; `/pay` no asienta (RN-3) | Coordinación con sistema externo. |
| Revelación de `direccion_texto` solo tras acuerdo/urgente (RN-6) | Control de autorización por contexto. |
| Verificación de cadena de hash de `reputation_log` (IN-8) | Job de auditoría que recomputa `hash_actual`. |
| `saldo ≥ 0` en billeteras usuario/empresa | Chequeo sobre la proyección antes de asentar. |
